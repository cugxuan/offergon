因为客户端发送和收到确认需要两次握手，服务端发送和收到确认也需要两次所以一共是 4 次。当第 2,3 次挥手合并的情况下，只需要三次挥手就可以完成断开。

       A                                B
       |                                |
       | (1) FIN ---------------------->|
       |                                |
       |                     (2) ACK    |
       |<-------------------------------|
       |                                |
       |                                |
       |                     (3) FIN    |
       |<-------------------------------|
       |                                |
       | (4) ACK ---------------------->|
       |                                |

- **四次挥手**：每个方向的连接都需要单独关闭，因此需要四次报文交换。每个方向各需要一个 FIN 和一个 ACK。
- **半关闭状态**：在发送 FIN 报文后的一段时间内，TCP 连接处于半关闭状态，一端可以不再发送数据，但仍然可以接收数据。
- **TIME_WAIT 状态**：主机A进入 `TIME_WAIT` 状态后，会等待一段时间（通常是 2 倍的最大报文生存时间，2MSL），以确保主机B收到了它的 ACK 报文并已经关闭连接。这个状态可以防止旧的重复数据包干扰新的连接。

### TIME_WAIT 状态的意义

- **TIME_WAIT**：主机A在收到主机B的 FIN 报文后，发送最后一个 ACK 报文，并进入 `TIME_WAIT` 状态。这个状态会持续一段时间（通常是 2 倍的最大报文生存时间，2MSL），以确保最后一个 ACK 报文能够到达主机B。如果主机B在这段时间内没有收到 ACK，它可以重新发送 FIN 报文。
- **防止旧连接数据干扰**：`TIME_WAIT` 状态还可以确保旧的重复数据包不会干扰新的连接。
