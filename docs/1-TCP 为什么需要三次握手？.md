1. **首要原因是为了避免历史连接，防止旧的重复连接初始化造成混乱。** 客户端在发起一次连接时，如果因为网络超时分别发送 (SYN = 90) 和 (SYN = 100) 的请求连接包，如果服务器都做出响应分别返回 (ACK = 91, SYN = 100) 和 (ACK = 101, SYN=110) 的回包，会造成两个连接建立。所以客户端应该对 (ACK = 101, SYN=110) 回复 (ACK=111)，而对 (ACK = 91, SYN = 100) 回复 RST 报文终止连接。
2. 同步双方序列号。如果没有第三次握手，服务器并不知道自己的哪个会话和客户端正确建立了连接。
3. 避免浪费资源。如果没有第三次握手，服务器收到 SYN 请求就需要回复 ACK，**并同时创建连接占用系统资源**，如果在丢包严重的情况下，收到多个超时的 SYN 等情况下会占用资源。
